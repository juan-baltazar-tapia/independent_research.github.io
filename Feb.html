 Numpy Arrays vs Lists 
Why use them in the first place instead of Python's built in array data structure? Well, Numpy arrays use less memory, are more efficient, and convenient when doing operations on large data sets. Here is an example of a Numpy array and Python List memory usage.

Numpy  Arrays Basics

Similar to the range() function in python, numpy has a arange() function where you can set the starting parameter, the ending parameter, and skip by certain amount parameter.

Lets take a look into the memory usage of a single integer in a numpy array. We observe that each integer is using 64 bits of data. 

Numpy allows us to change how many bits this integer array uses by setting dtype (data type) to int8. One downside of using numpy arrays is that they are homogenous, meaning each numpy array can only contain the same datatype.

N-Dimensional Arrays

You can also create higher dimensional arrays, and use the shape method to find the number of columns, and number of rows

The reshape method is used to change the number of rows and columns, such that the column and rows perfectly divide the orignal total length. For example, I could've have reshaped to 6 columns and 1 row as well.

You can create empty arrays with zeros or ones. By defualt they are floating type.

You can alter the arrays by selecting as shown below, you can also select different elements using different operators such as less than, greater than, etc. You can also use the python's slice operations.

Sorting Numpy Arrays

Sorting Numpy arrays is as simple as using the sort method. 

You can even change the sorting algorithm that is used. By default, numpy sort uses quicksort

There are two ways to copy an array using NumPy. There is the .view method, and .copy method

The view method will modify the original array. It can be usefull at times, or not, so be carefull. Instead, use the copy method, which won't change the original array

Ravel vs Flatten methods

What if instead of rows and columns, you want a single one dimensional array? You can do that using the ravel and flatten functions. In the same way the view method affected the orignal arrray, and copy didn't, the ravel function makes a view of the array, and thus effects the orignal array, while the flatten method makes a copy of the orignal array.

In summary,  flatten method => copy = modifes original and ravel method => view => modifes original

KNeighborsClassifiers - algorithm parameter

There are three algorithms you can specify to use, by defualt it is set to auto, which automatically determines which algorithm to use. First, the brute force algorithm

The brute force algorithm computes the distance between a new instance and all instances in the training set, sorts distances, selects the k nearest neighbors, and does a majority vote.
Time complexity: O(k * n * d) where n is the number of points in the training dataset, d is the data dimesnitonality, and k is number of neighbor's.

k-d tree method and ball tree method both organize the training points from X into a tree data structure, that allows for faster nearest neighbors lookup.
In summary, these methods create a tree data strucutre that splits the data into k-dimensions of space, (k here being k dimensions). They have training time and space complexity, unlike the brute force approach.
Prediction time complexity: O(k * log(n))
Prediction time space Complexity: O(1)


p value, power parameter for the Minkowski metric

When p = 1, manhattan distance is used. Manhattan distance is less computational intensive
When p = 2, euclidean distance is used. The minkowski distance is equal to the manhattan distance when p = 1, and equal to the euclidean distance when p = 2. For any other number, the minkowski distance is used
